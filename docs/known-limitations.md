# Known Limitations

## Current Limitations

- **WebGPU evidence is adapter-specific**: Intel hardware currently passes reductions, 4D full-spectrum/CLV/FTLE, and the 6D N-chain STM/QR gate. The vendor matrix remains partial until physical NVIDIA and AMD runners upload passing ladder artifacts; software adapters are not accepted.
- **N-chain GPU is a hybrid variational path**: for planar chains up to N=8, CPU f64 integrates the nonlinear reference trajectory and constructs the central-difference Jacobian tape. WebGPU f32 performs tiled STM propagation, QR tape, Ginelli backward solves, and the singular-value FTLE estimate. A same-tape CPU f64 oracle controls promotion. Full nonlinear N-chain trajectory/Jacobian generation on GPU is not claimed.
- **Visual regression golden baselines**: Chromium/Windows `toHaveScreenshot` baselines are version-controlled for `npm run test:visual`. The Ubuntu mainline CI keeps visual snapshots out of its default browser gate until Linux baselines are generated on a Linux runner and committed.
- **MATLAB / Julia cross-checks optional and unpinned**: the Python/SciPy/SymPy stack is pinned via `requirements.txt` and `Dockerfile`; MATLAB and Julia comparisons remain manual and unpinned.
- **Browser memory metrics Chromium-only**: `performance.memory` is a Chromium extension; Firefox/WebKit report `undefined`.
- **Non-commutative stochastic noise**: Lévy-area correction terms are required for strong order ≥ 1 under non-commutative matrix diffusion; `commutativeMilsteinStep` is only strong order 1 in the commutative case. This is no longer silent — `commutativityDefect` (`src/physics/noiseCommutativity.ts`) measures the Lie-bracket obstruction `max_{i,j<k}|L_jB_{i,k}−L_kB_{i,j}|`, so callers can detect a non-commutative `B` (defect > 0) before trusting the scheme.
- **Polar-chart singularity in spherical chain**: the `(θ, φ)` chart is clamped at `|sinθ| = 1e-6`; trajectories passing exactly through a pole are perturbed at that scale. Use `EmbeddedSphericalChain` for near-pole fidelity.
- **NS invariant-circle collocation at Arnold tongues**: the trigonometric-collocation solver requires smooth conjugacy to rigid rotation; phase-locked parameters inside Arnold tongues are out of scope for the *collocation* curve. `continueNeimarkSackerTorusRobust` (`src/chaos/arnoldTongue.ts`) now falls back to the directly-measured orbit-winding rotation number there (valid mode-locked or quasi-periodic), tagging each point's `rotationNumberSource`, so the rotation number is still reported inside a tongue even though the smooth curve is not.
- **Arclength continuation beyond the first bifurcation**: `continueArclength` traces folds but does not switch onto bifurcated branches; following branched-off solutions after a detected bifurcation requires `switchPeriodDoubling`, `switchSymmetryBreaking`, or `switchTranscriticalBranch` and a manual starting point.
- **webkit / Windows audio in e2e**: `modern-audio.spec.ts` can fail on Windows when the host audio backend stalls headless WebKit's `AudioContext`. This is an environment issue; CI (Linux) is unaffected.
- **CSP and inline scripts**: future UI patches must use module files and event listeners; new `innerHTML` patterns will be blocked by the existing Content Security Policy.

## Numerical accuracy (recent upgrades)

- **Variational flow now uses an exact analytic Jacobian** for the double pendulum (`jacobianDouble`), removing the ~1e-7 error floor that a finite-difference Jacobian imposed on the full Lyapunov spectrum, SALI and FLI. Systems without a closed-form Jacobian fall back to an O(h^2) **central-difference** Jacobian (previously forward-difference). Both are checked against each other in `tests/analytic-derivatives.test.ts`.
- **The canonical implicit-midpoint integrator is now genuinely symplectic**: `hamiltonianGradient` is the exact analytic gradient of H(q,p) rather than a central-difference approximation, so the method integrates the true Hamiltonian vector field.
- **Lyapunov estimates now report uncertainty** (standard error and an approximate 95% CI from the converged per-interval local exponents). The naive SE ignores autocorrelation between renormalization intervals and is a lower bound; a **batched-means / block-bootstrap standard error** (`batchedStandardError`, surfaced as `blockStdError` on the result and CSV export) decorrelates neighbouring intervals for a more honest uncertainty.
- **The analytic Jacobian is validated against an external closed form**, not just another finite difference: linearised about the downward equilibrium, the lower-left block of `jacobianDouble` reproduces the textbook equal-pendulum normal-mode frequencies ω² = (2 ± √2) g/l to machine precision (`tests/normal-modes.test.ts`).
- **Integrator orders are empirically certified.** `empiricalOrder` (`src/research/convergenceOrder.ts`) measures the convergence order by Richardson self-convergence — log₂‖y(dt)−y(dt/2)‖/‖y(dt/2)−y(dt/4)‖ — confirming euler≈1, rk2≈2, rk4≈4, gauss2≈4 with no analytic reference (`tests/convergence-order.test.ts`).
- **Structure preservation is empirically certified over long runs.** `energyDriftProfile` (`src/research/structurePreservation.ts`) integrates a conservative system for thousands of periods and classifies the relative-energy drift as *secular* (a monotone trend, high-R² line) or *bounded* (an oscillation within a band). On the planar double pendulum (~1000 periods) the non-symmetric **rk4 drifts secularly** (R² = 0.99999, |drift| growing past 2·10⁻⁴) while the time-symmetric **2-stage Gauss collocation `gauss2` stays bounded** (R² ≈ 0.19, |drift| ≲ 7·10⁻⁸ with no trend) and **implicit-midpoint `hmidpoint`** likewise — the defining structure-preserving signature, turning "use a symplectic integrator for long conservative runs" into a measured number (`tests/structure-preservation.test.ts`; CLI `npm run research -- drift`). The same `method` selection is exposed on the chain integrators (`SphericalChain({ method: 'gauss2' })`); for the chains in (θ, ω) coordinates the bounded-drift mechanism is the method's time-reversibility rather than strict canonical symplecticity (which would need (q, p) coordinates — the route the canonical implicit-midpoint integrator already takes).
- **The Lyapunov spectrum is checked for Hamiltonian self-consistency.** `analyzeSpectrumConsistency` verifies the conservative-system constraints (Σλ ≈ 0, symplectic pairing λ_i ≈ −λ_{n−1−i}, two zero exponents) and the verdict is attached to every spectrum result and shown in the Lyapunov tab status line — an independent gate on the whole tangent-space pipeline rather than a bare number.
- **Lyapunov runs accept a selectable integrator** (defaulting to rk4); a high-order/symplectic reference can be chosen for long conservative runs.
- **Shadowing / reproducibility horizon** (`shadowingHorizon`) quantifies how long a chosen integrator tracks a high-accuracy (gbs, fine-dt) reference before the chaotic divergence exceeds a tolerance — turning numerical trust into a measured number.

## Chaos diagnostics (research-grade additions)

- **Independent chaos detector: the 0–1 test** (`zeroOneTest`, Gottwald–Melbourne). It works on a scalar observable series alone and never touches the Jacobian or a deviation vector, so K ≈ 1 (chaotic) / K ≈ 0 (regular) is a genuine cross-validation of the Lyapunov verdict rather than the same tangent-space computation reported twice. Confirmed against periodic/white-noise series and both ends of the double pendulum (`tests/zero-one-test.test.ts`).
- **Covariant Lyapunov vectors (Ginelli algorithm)** (`covariantLyapunovVectors`). These are the true Oseledets stable/unstable directions — covariant and generally non-orthogonal — not the orthogonal Gram–Schmidt frame. Correctness is pinned on a linear system whose CLVs are known non-orthogonal eigenvectors (a Gram–Schmidt implementation would fail this), and the QR-diagonal exponents cross-check `lyapunovSpectrum`. A hyperbolicity measure (the minimum angle between expanding and contracting CLVs) is reported (`tests/clv.test.ts`).
- **Fractal exit structure: flip basins, basin entropy, box-counting dimension** (`doublePendulumFlipBasin`, `basinEntropy`, `boxCountingDimension`). The classic double-pendulum flip map is classified, and its boundary is quantified by the Daza et al. basin entropy and a Minkowski–Bouligand dimension (≈1.5–1.6 — strictly fractal). The entropy/box-counting routines are pure functions of a label grid, so they apply equally to genuine attractor basins (`tests/basin.test.ts`).
- **Recurrence Quantification Analysis (RQA)** (`recurrenceQuantification`, `recurrenceMatrix`; Marwan et al., Phys. Rep. 2007). A second non-variational diagnostic: a bounded observable is time-delay embedded and the recurrence plot is quantified into DET (determinism — structure vs noise), LAM, Lmax and DIV = 1/Lmax (a finite-size proxy for the largest Lyapunov exponent), ENTR and trapping time. Pinned on a sine (DET ≈ 1, long diagonals) vs white noise (low DET), and the double-pendulum chaos signature (a regular orbit has far longer diagonals / smaller divergence than a chaotic one) cross-validates the spectrum (`tests/rqa.test.ts`).
- **Finite-Time Lyapunov Exponent (FTLE) fields** (`finiteTimeLyapunov`, `flowMapGradient`, `doublePendulumFtleField`). σ_T(x₀) = (1/T) ln σ_max(∂x(T)/∂x(0)) — the largest singular value of the *finite-time* flow-map gradient, obtained by propagating the variational state-transition matrix (same analytic Jacobian, no finite-difference floor). Ridges of the field are Lagrangian Coherent Structures, distinct from the Sweep's time-averaged λ₁. Pinned against the exact `exp(AT)` of linear systems (FTLE and determinant analytically known) and the double-pendulum chaotic-vs-regular separation (`tests/ftle.test.ts`). The single-largest singular value is found by power iteration on MᵀM.
- **Floquet stability of periodic orbits** (`floquetAnalysis`, `drivenPeriodicOrbit`, `eigenvalues2x2`, `monodromyMatrix`). The monodromy matrix over one period (the same state-transition matrix as the FTLE flow-map gradient) has the Floquet multipliers ρ_i as eigenvalues; |ρ_i| < 1 ⇒ a stable orbit. For the driven pendulum a period-1 orbit is found as a fixed point of the stroboscopic map by a 2-D Newton step whose Jacobian is the monodromy itself. Planar (2×2) scope, so the eigenvalues are closed-form (no general eigensolver). Pinned on linear oscillators whose monodromy determinant is analytically e^{∫div} — undamped (unit circle), damped (Π|ρ| = e^{−2γT}, stable), inverted (a multiplier outside the unit circle, unstable) — and the driven-pendulum stroboscopic orbit (`tests/floquet.test.ts`). **Now surfaced in the Bifurcation tab** ("Driven Orbit: Floquet & Pitchfork" → *Floquet @ A*), branch-traced from A = 0.7 so the multipliers track the symmetric oscillating branch; cross-browser e2e in `e2e/modern-bifurcation-tab.spec.ts`.
- **Numerical continuation + bifurcation detection** (`continueDrivenPeriodicOrbit`, `classifyBifurcation`). Natural-parameter continuation traces the driven-pendulum period-1 orbit across a parameter (warm-starting Newton from the previous orbit) and reports the first stability loss, classified by the Floquet crossing: ρ → −1 period-doubling, ρ → +1 tangent (fold/transcritical/pitchfork — reported generically), a complex pair through |ρ| = 1 Neimark–Sacker. Validated on constructed multiplier sets and on the damped driven pendulum, whose symmetric period-1 orbit loses stability via a real multiplier crossing +1 near drive amplitude A ≈ 1.01 (`tests/continuation.test.ts`). Library API.
- **Pseudo-arclength continuation with fold detection** (`continueArclength`, Keller's method). Natural-parameter continuation stalls at a fold (the branch has a vertical tangent there); pseudo-arclength advances along the branch's arclength, solving the augmented system [G(x,λ)=0; τ·Δ−ds=0] whose Jacobian (last row τ) stays non-singular through the turning point, so it traces *around* folds. The Jacobians are finite-differenced, so it is generic over any residual G(x,λ) (algebraic, or the stroboscopic fixed-point condition of the driven pendulum). Pinned on branches with closed-form folds: x²−λ (one fold at the origin) and x³−3x−λ (folds at (1,−2) and (−1,2)), staying on the solution curve to ~1e-10 and locating the folds to the step granularity (`tests/arclength.test.ts`). Library API.
- **Every non-variational diagnostic now carries an uncertainty estimate**, extending the Lyapunov block-bootstrap SE to the rest of the toolbox: the 0–1 test reports a seeded bootstrap SE and percentile 95% CI of the median K (the per-frequency K_c are i.i.d. over random frequencies, so the ordinary bootstrap is the right scheme); RQA reports block-resampled SEs for DET/DIV (contiguous blocks preserve the temporal recurrence structure — batched-means style — and cost ~1/k of the O(N²) quantification); basin entropy reports the SEM of Sb/Sbb over boxes; and the box-counting dimension reports the regression slope SE, Student-t 95% CI and R² of the log-log fit (a low R² flags a dimension that should not be trusted). All surfaced in the 0–1/RQA/Basin tab readouts and status lines (`tests/uncertainty.test.ts`).
- **Literature anchors** (`runLiteratureAnchors`, `npm run validate:literature`, CI job): engine-computed quantities are compared head-to-head against values that exist *outside* the codebase — the large-amplitude pendulum period vs 4K(sin(θ₀/2))/ω₀ (complete elliptic integral, agreement < 1e-6), the equal-double-pendulum normal modes (2 ∓ √2)g/l, the Melnikov A_c by quadrature vs closed form, and the damped driven pendulum period-doubling onset measured from the Floquet multiplier crossing −1: **A_PD = 1.06637 measured vs 1.0663 published** (Baker & Gollub). Structural checks: A_c < A_PD (the homoclinic tangle precedes the attractor cascade) and the flip-basin boundary dimension is strictly fractal. Report: `reports/literature-anchors.md`; tests: `tests/literature-anchors.test.ts`.
- **Melnikov analysis — the analytic chaos threshold** (`melnikovCriticalAmplitude`, `melnikovFunction`, `melnikovVerdict`; CLI `npm run research -- melnikov`). For the damped driven pendulum the Melnikov function along the unperturbed separatrix evaluates in closed form, M(τ₀) = −8δ + 2πf·sech(πΩ/2)·cos(Ωτ₀), giving the critical drive amplitude A_c = (4γω₀/π)·cosh(πω/(2ω₀)) above which a transverse homoclinic intersection (Smale-horseshoe tangle) exists. The closed form is pinned against brute-force Simpson quadrature of the Melnikov integral along the separatrix (≤1e-8), the known value A_c ≈ 1.0187 for γ = 0.5, ω = 2/3, and 0–1-test physics consistency: regular motion well below A_c, and the literature chaotic preset (A = 1.15) above it (`tests/melnikov.test.ts`). Being first-order perturbation theory, A_c is asymptotically exact only for small damping/forcing; the tangle it predicts yields transient chaos and fractal basin boundaries, with the sustained chaotic attractor appearing somewhat above A_c (the measured period-doubling onset A_PD ≈ 1.066 indeed sits above A_c ≈ 1.019 — the expected ordering).
- **Five of these are first-class Lab tabs**, not just library APIs: the 0–1 Test (◐), Covariant Lyapunov Vectors (⇌), Flip Basins (❋), Recurrence Quantification (▨) and FTLE Field (⩜) tabs each run their job on the chaos worker (with the same transparent main-thread fallback as the Lyapunov tab) and read the live on-page system controls via `readSystem()`. The worker path is exercised by `tests/chaos-protocol-diagnostics.test.ts` (the same pure `runChaosJob` handler the UI uses) and the tab takeovers by `e2e/modern-chaos-diagnostics.spec.ts`. The rail icons are grouped into Simulate / Analyze / Chaos-diagnostics / Validate clusters (subtle dividers, scroll-safe).

## Spherical chain, symbolic reference, and the mini-paper (v10.30)

- **The spherical double/triple pendulum** (`src/physics/sphericalChain.ts`) is genuine 3D chain dynamics (ball joints, 4/6 DOF), not a camera over planar motion. Its equations are assembled in manipulator form from closed-form Jacobians; validity is pinned by exact reduction to the single spherical pendulum and to the planar chain, by E and L_z conservation in 3D chaos (with 4th-order drift shrinkage under dt-halving — a wrong RHS would leave a dt-independent floor), and by the independent SymPy symbolic derivation (~1e-14). Known limitation: the (θ, φ) chart is singular at the poles; it is regularised at |sinθ| = 1e-6, so trajectories that pass *exactly* through a pole are perturbed at that scale. Damping is per-coordinate rate damping (matching `spherical.ts`), which differs from the planar chain's force-level convention — planar comparisons are therefore made at γ = 0. For the **single** spherical pendulum this pole limitation is now removed by an alternative **embedded (Cartesian unit-vector) chart** (`EmbeddedSphericalPendulum`, `src/physics/sphericalEmbedded.ts`): the ambient member of the S² atlas, globally regular, with the rod constraint enforced by projection. It agrees with the (θ, φ) chart to ~2e-10 away from the poles and conserves E and L_z to ~1e-11 through a near-pole passage where the clamped polar chart loses ~30% of the energy (or diverges to NaN) — `tests/spherical-embedded.test.ts`. **The embedded *chain* is now implemented too** (`EmbeddedSphericalChain`, `src/physics/sphericalEmbeddedChain.ts`): each link is an independent Cartesian unit vector u_k ∈ S² with the rod constraints enforced by N Lagrange multipliers (the rod tensions). The decisive structural point is that the kinetic metric in the embedded chart is the *constant* suffix-mass matrix A_{jk} = l_j l_k S_{max(j,k)}, so there are **no configuration-dependent Coriolis/Christoffel terms** — all nonlinearity is in the constraint forces, and the per-step multiplier system G λ = c has G = B∘U (Hadamard product of the SPD inverse-metric with the unit-diagonal Gram matrix of the link directions), which is **positive-definite for every configuration by the Schur product theorem**. Unlike the polar chain there is therefore no singular solve and no pole clamp anywhere. Validated by exact reduction to `EmbeddedSphericalPendulum` at N = 1 (RHS component-wise and ~7e-10 over a 3 s trajectory), agreement with the polar `SphericalChain` to ~1.8e-7 away from the poles, E/L_z conservation to ~1e-5 through a near-pole passage where the clamped polar chart drifts > 10× more, 4th-order drift shrinkage under dt-halving, and a positive-definite multiplier matrix at a state with a link essentially *at* the pole (`tests/spherical-embedded-chain.test.ts`). Library API; the natural future step is to surface it in the 3D lab.
- **The SymPy reference** (`npm run validate:sympy`) checks the *derivation* (RHS component-wise at random states), deliberately not trajectories — the SciPy DOP853 comparison already covers integration. It validates conservative dynamics only (γ = 0), since the damping conventions are engine choices that a Lagrangian derivation does not fix.
- **The mini-paper** (`paper/`) measures A_PD(γ) for the *primary oscillating attractor branch*, warm-started in A. At the lowest dampings the phase space is multistable and basin-capture transitions can occur below the verified doubling (observed at γ = 0.15); the quoted onsets are specifically Floquet-verified ρ = −1 crossings. The Melnikov comparison uses the first-order closed form; its breakdown at strong damping is part of the paper's result, not an engine defect.

- Covariant Lyapunov vectors (Ginelli) and finite-time Lyapunov (FTLE) fields are now implemented and surfaced as Lab tabs; the full spectrum, CLVs and variational FTLE are CPU-side publication oracles. GPU acceleration has an executable promotion contract (`compareLyapunovSpectrumAcceleration`, `compareClvAcceleration`, `compareFtleFieldAcceleration`) and must pass that contract before replacing a CPU result. The 0–1 test, CLV, basin-entropy, RQA and FTLE diagnostics are dedicated Lab UI tabs (in addition to their library/test APIs); Floquet stability and the symmetry-breaking pitchfork are now surfaced in the **Bifurcation tab** (in addition to the library and CLI).
- A **double-double (~31-digit) extended-precision reference** is now available (`src/validation/doubleDouble.ts`): error-free-transform arithmetic (two-sum / two-product, Dekker–Knuth), double-double division, and double-double sin/cos (argument-reduced Taylor series, accurate to dd precision — sin²+cos² = 1 holds to ~1e-29 where float64 manages only ~1e-16), plus double-double RK4 and a time-symmetric Störmer–Verlet integrator. A symmetric forward/backward round trip shows ~17 orders of magnitude less round-off than float64, and the double-double RK4 acts as an exact-arithmetic reference that measures the float64 integrator's accumulated round-off (`tests/double-double.test.ts`).
- The double-double arithmetic is now applied to the **double pendulum itself** (`src/validation/doubleDoublePendulum.ts`): `ddDoublePendulumRhs` re-expresses the mass-matrix solve of `rhsDouble` in double-double and is validated component-wise against the trusted float64 `rhsDouble` to ~1e-15. Using it as the ground truth, the float64 RK4 orbit is shown to track the true trajectory only briefly (~1e-14 at T = 2 s) before the chaotic dynamics amplify its round-off by orders of magnitude (~1e-7 at T = 10 s) — a predictability limit an *in-precision* reference cannot expose about itself (`tests/double-double-pendulum.test.ts`). This closes the earlier gap where the pendulum's only reference was a high-accuracy in-precision orbit. (The reference is conservative/undamped.)
- **External cross-validation against SciPy is now automated** (`npm run validate:cross`, also a CI job): `scripts/scipy_reference.py` re-derives the equations of motion independently and integrates them with `solve_ivp` DOP853 at rtol = atol = 1e-13, and the TypeScript engine integrates the same initial conditions with its own RHS (RK4, dt = 2e-5). **Both the double and the triple pendulum are covered** — the double against closed-form Lagrangian equations, the triple against the general N-chain mass-matrix formulation solved with `numpy.linalg.solve` (vs the engine's hand-expanded 3×3 elimination — a different derivation *and* linear-solve path). Measured agreement: double — regular ~4e-14 over 20 s, chaotic ~6e-11 at T = 10 s; triple — regular ~6e-14 over 20 s, chaotic ~4e-8 at T = 8 s. All consistent with the shared tolerance floor amplified by e^{λ₁t}, which is exactly the expected behaviour of two *correct* independent implementations of the same chaotic system. Report: `reports/cross-validation.md`. (MATLAB/Julia cross-checks remain optional future additions.)
- A **Wada-property candidate test** (`wadaCandidate`, grid method of Daza–Wagemakers–Sanjuán) classifies what fraction of basin-boundary cells touch ≥ 3 basins; it is reported in the Basin tab status line and refinable from the CLI (`npm run research -- wada --n 150`). At a single resolution it is *evidence*, not proof — the verdict field is deliberately named `wadaCandidate`.

- The **figure pack export** captures every drawn analysis canvas as a captioned PNG (blank canvases are skipped — a tab must have rendered at least once to be included) into a print-to-PDF-friendly HTML gallery; canvases are raster, so SVG export is not offered.
- **Bifurcated-branch following is now implemented for the codimension-1 cases**: period-doubling (`switchPeriodDoubling`, `drivenPeriodicOrbitN`), symmetry-breaking pitchfork (`switchSymmetryBreaking`), transcritical branch switching (`switchTranscriticalBranch`), and Neimark-Sacker torus continuation (`continueNeimarkSackerTorus`). At a detected PD (real Floquet multiplier through −1) the period-2 orbit is found by Newton on the *double-period* stroboscopic map, seeded along the critical eigenvector of the monodromy, with a separation check so falling back onto the period-1 point is rejected rather than misreported. Verified on the classic damped driven pendulum (γ = 0.5, ω = 2/3): the multiplier crosses −1 between A = 1.065 and 1.07 (literature A_PD ≈ 1.0663) and the switched period-2 orbit is stable — the attractor direct simulation shows just past onset (`tests/branch-switching.test.ts`; CLI `npm run research -- switch`). Remaining branch-following work is broader high-dimensional continuation, automated branch selection, and robust UI orchestration across multiple competing bifurcations.
- **A figure-rich research notebook is generated by `npm run notebook`** (`reports/research-notebook.html`): headline numbers computed through the same `runChaosJob` handler the worker/CLI/tests share, and figures captured from the *running* application (each analysis tab driven to completion headlessly). Self-contained HTML, print-to-PDF friendly.
- **The inverse problem is now supported** (`fitDoublePendulum`, `levenbergMarquardt` in `src/research/parameterEstimation.ts`): physical parameters are recovered from an observed angle trajectory by nonlinear least squares with the engine's own RHS in the fit loop, and the fit reports **parameter uncertainty** (covariance s²·(JᵀJ)⁻¹, standard errors, correlation, degrees of freedom). It is deliberately scoped to short observation windows — over long horizons the chaotic forward map is exponentially ill-conditioned, so the Jacobian degrades and the basin of attraction shrinks. Estimating the full (m₁, m₂, l₁, l₂, g) set jointly is also not always identifiable from angles alone (scaling degeneracies); fit an identifiable subset. For a nonlinear model the reported covariance is the standard linearised approximation; for a linear-in-parameters model it is exact (validated by Monte Carlo). `tests/parameter-estimation.test.ts`. **Now surfaced in the Research+ Lab tab** ("Recover g" — recovers g ≈ 9.81 from a synthetic trajectory with its standard error; cross-browser e2e `e2e/modern-research-plus.spec.ts`) alongside the headless CLI: `npm run research -- estimate --estimate g,l2`.
- **Stochastic forcing supports additive *and* multiplicative noise** (`runLangevinEnsemble`, `eulerMaruyamaStep`, `milsteinStep`, `src/physics/stochastic.ts`), validated against the Brownian, Ornstein–Uhlenbeck, and Geometric-Brownian-Motion closed forms. Euler–Maruyama (strong order ½) and Milstein (strong order 1, with the b·b′ correction) are provided for diagonal noise. **Full matrix-diffusion schemes are now implemented and wired into the ensemble runner**: `stochasticHeunStratonovichStep` (a predictor–corrector for Stratonovich SDEs dx = a dt + B(x)∘dW, so multiplicative noise is not silently integrated under the Itô convention) and `commutativeMilsteinStep` (strong order 1 for the commutative-noise case, with the Lie-derivative correction L_j B_{i,k}); both validated against the closed forms (Heun reproduces the Stratonovich predictor–corrector formula and Brownian variance σ²t; commutative Milstein reduces to the diagonal-GBM Itô moments on a diagonal B). Non-commutative noise still needs Lévy-area terms and is intentionally not approximated. **Adaptive (step-size-controlled) SDE integration is also implemented** (`runAdaptiveLangevinPath` over a frozen dyadic `buildBrownianGrid`): the Wiener path is sampled once on a fine grid so the integrator can take dyadic-multiple steps whose increments are exactly consistent under refinement (no Brownian-bridge resampling), with step-doubling local-error control. Validated by exact reduction to an adaptive Euler ODE at σ = 0 (tracks e^{−t}, coarsening below the fine grid), reproducibility for a fixed grid seed, and pathwise (strong) convergence to the all-fine Milstein solution on the *same* Brownian path (`tests/stochastic.test.ts`). **Surfaced in the Research+ Lab tab** ("Run ensemble" — the double pendulum under a stochastic torque, with a log-variance plot) and the headless CLI `npm run research -- sde --sigma 0.5 --scheme milstein` (diagonal-noise ensemble); the matrix/adaptive schemes are library API.
- **Symmetry-breaking pitchfork branch following is implemented** (`switchSymmetryBreaking`): the +1-crossing case, where the symmetric driven-pendulum orbit gives birth to two mirror-image asymmetric orbits, is now followed (verified by the midpoint Z₂ signature), exposed on the headless CLI (`npm run research -- pitchfork`), and surfaced in the **Bifurcation tab** ("Find pitchfork", cross-browser e2e). **Transcritical (+1) switching is now implemented too** (`switchTranscriticalBranch`): a generic Newton switch for algebraic/fixed-point residuals that steps the parameter off the crossing along the supplied other-branch tangent and rejects solutions that fall back onto a reference branch — verified on the normal form r(x, λ) = λx − x² (branches x = 0 and x = λ exchanging stability at λ = 0), where the switch lands on x = λ to ~1e-10 (`tests/branch-switching.test.ts`; CLI `npm run research -- transcritical`). Every codim-1 branch-following case (period-doubling −1, pitchfork +1, transcritical +1, and the NS torus) is now covered.
- **Neimark–Sacker torus continuation is now implemented** (`continueNeimarkSackerTorus`, `src/chaos/neimarkSacker.ts`): the invariant closed curve born at an NS bifurcation of a 2D stroboscopic map is computed by **trigonometric collocation** of the invariance equation F(u(θ)) = u(θ + 2πρ) — damped Newton on the M curve samples and the rotation number ρ together, with a Dirichlet-kernel rotation operator and a Poincaré phase condition — and continued in the parameter. It is validated on the **delayed-logistic map** (the textbook NS example: onset at a = 2 with ρ → 1/6) where ρ → 1/6 and the curve amplitude → 0 monotonically toward onset, on a pure-rotation map (exact, ρ = α/2π), and — independently of the solver's own residual — by iterating the *raw* map from a point on the computed curve and confirming the orbit stays on the curve (`tests/neimark-sacker-torus.test.ts`). Library API. The reported `invarianceResidual` is sampled *between* collocation nodes, so it is a genuine truncation error rather than the Newton residual; phase-locked (Arnold-tongue) parameters where conjugacy to rigid rotation fails are out of scope and flagged in the caveat. **Now surfaced in the Bifurcation tab** ("Neimark–Sacker invariant circle" → *Trace invariant circle*): the family of closed curves is continued from a = 2.05 toward onset and drawn on canvas, shrinking onto the fixed point with ρ → 1/6 (cross-browser e2e `e2e/modern-bifurcation-tab.spec.ts`).

  - **Arnold tongues / phase-locking** (`scanModeLocking`, `rotationNumber`, `planarMapRotationNumber`, `sineCircleMap`, `src/chaos/arnoldTongue.ts`): the rotation number is measured *directly* by lift/orbit winding — valid mode-locked *or* quasi-periodic, where the collocation solver is not — so the parameters the NS solver declares out of scope become a detectable feature. Pinned on the sine circle map (the textbook Arnold-tongue system): ρ = Ω exactly at K = 0, and at K = 1 the 1/2 tongue is located with finite width (Ω ≈ [0.46, 0.54]) and ρ(Ω) is a monotone devil's staircase within the estimator's O(1/N) noise; the same winding estimator on the delayed-logistic invariant circle reproduces the solver's ρ → 1/6 (`tests/torus-analysis.test.ts`; CLI `npm run research -- arnold`).
  - **Lyapunov spectrum on the torus** (`torusLyapunovSpectrum`): a Benettin/QR spectrum on the 2-D stroboscopic map restricted to the invariant circle. A quasi-periodic torus is *neutral* — the largest exponent is ≈ 0 (the on-circle direction; in the underlying flow it is one of the 2-torus's two zero exponents), with the transverse exponent < 0 for an attracting circle — so a strictly positive value would falsify the smooth-torus picture. This is an independent self-consistency gate mirroring the Hamiltonian-spectrum check: pure rotation gives two ≈ 0 exponents, the delayed-logistic torus gives [≈0, −0.021] and a `quasi-periodic-torus` verdict (CLI `npm run research -- toruslyap`).

This is the NS analogue of the Richardson `empiricalOrder` integrator-order certification.
