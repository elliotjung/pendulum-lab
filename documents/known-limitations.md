# Known Limitations

## Current Limitations

- **WebGPU evidence is adapter-specific**: Intel hardware currently passes reductions, 4D full-spectrum/CLV/FTLE, and the 6D N-chain STM/QR gate. The vendor matrix remains partial until physical NVIDIA and AMD runners upload passing ladder artifacts; software adapters are not accepted.
- **SharedArrayBuffer transport is capability-gated**: the tested ring-buffer primitive falls back to local memory, while side plots still use transferable snapshots. Promotion requires the configured Cloudflare mirror to prove COOP/COEP and cross-origin isolation; GitHub Pages cannot set those headers.
- **Double-double arithmetic is validation-only**: `src/validation/doubleDouble.ts` and the double-double pendulum helpers are for short-horizon reference checks and numerical audits. They are intentionally excluded from real-time UI loops and should not be used for interactive Lab rendering.
- **N-chain GPU is a hybrid variational path**: for planar chains up to N=8, CPU f64 integrates the nonlinear reference trajectory and constructs the central-difference Jacobian tape. An ABI-2 AssemblyScript f64 SIMD candidate can accelerate that CPU-side tape in headless tests, but reports `promoted: false` until cross-engine and repeatable-speedup evidence is complete. WebGPU f32 performs tiled STM propagation, QR tape, Ginelli backward solves, and the singular-value FTLE estimate. A same-tape CPU f64 oracle controls promotion. Full nonlinear N-chain trajectory/Jacobian generation on GPU is not claimed.
- **Synchronization/chimera scope**: the Kuramoto/Huygens module is a phase reduction, not a full escapement/contact model of coupled clocks. `chimeraDiagnostics` reports finite-size coexistence of high/low local order as a candidate; it is not a thermodynamic-limit proof.
- **Dry friction scope**: Coulomb/Stribeck friction is smoothly regularized through zero velocity. It dissipates correctly but does not solve the exact static-friction complementarity cone.
- **Delay solver scope**: the Pyragas solver is fixed-step method-of-steps RK4 with linear accepted-history interpolation and requires `delay >= dt`. Delay-stability boundaries require dt refinement.
- **Recorded video core, not live camera UX**: colour-centroid marker tracking and angle conversion are headless/tested, but media permissions, calibration UI, occlusion recovery, and live webcam capture remain browser-product work.
- **Cycle expansion is explicitly truncated**: the periodic-orbit database removes cyclic duplicates and subperiodic roots and returns Euler-product/observable coefficients. The scalar observable is a leading prime-weight approximation; convergence must be established by coefficient decay and max-period refinement.
- **Visual regression golden baselines**: Chromium/Windows `toHaveScreenshot` baselines are version-controlled for `npm run test:visual`. The Ubuntu promotion workflow now validates a complete six-image set and reruns it without update mode; mainline automatically enables the Linux gate after those reviewed images are committed. Until that external Actions run and PR review occurs, Linux (and macOS) pixel baselines are not claimed. See `documents/visual-baseline-promotion.md`.
- **MATLAB remains optional**: the Python/SciPy/SymPy stack is pinned via `requirements.txt` and `Dockerfile`, while Julia 1.10.11 is pinned by `julia/Project.toml` plus `julia/Manifest.toml` and is a required mainline gate. MATLAB stays manual because its licensed runtime is not available on the public runner.
- **Browser memory metrics Chromium-only**: `performance.memory` is a Chromium extension; Firefox/WebKit report `undefined`.
- **Non-commutative stochastic noise**: LГ©vy-area correction terms are required for strong order вүҘ 1 under non-commutative matrix diffusion; `commutativeMilsteinStep` is only strong order 1 in the commutative case. This is no longer silent вҖ” `commutativityDefect` (`src/physics/noiseCommutativity.ts`) measures the Lie-bracket obstruction `max_{i,j<k}|L_jB_{i,k}вҲ’L_kB_{i,j}|`, so callers can detect a non-commutative `B` (defect > 0) before trusting the scheme.
- **Polar-chart singularity in spherical chain**: the `(Оё, ПҶ)` chart is clamped at `|sinОё| = 1e-6`; trajectories passing exactly through a pole are perturbed at that scale. Use `EmbeddedSphericalChain` for near-pole fidelity.
- **NS invariant-circle collocation at Arnold tongues**: the trigonometric-collocation solver requires smooth conjugacy to rigid rotation; phase-locked parameters inside Arnold tongues are out of scope for the _collocation_ curve. `continueNeimarkSackerTorusRobust` (`src/chaos/arnoldTongue.ts`) now falls back to the directly-measured orbit-winding rotation number there (valid mode-locked or quasi-periodic), tagging each point's `rotationNumberSource`, so the rotation number is still reported inside a tongue even though the smooth curve is not.
- **Arclength continuation beyond the first bifurcation**: `continueArclength` traces folds but does not switch onto bifurcated branches; following branched-off solutions after a detected bifurcation requires `switchPeriodDoubling`, `switchSymmetryBreaking`, or `switchTranscriticalBranch` and a manual starting point.
- **webkit / Windows audio in e2e**: `modern-audio.spec.ts` can fail on Windows when the host audio backend stalls headless WebKit's `AudioContext`. This is an environment issue; CI (Linux) is unaffected.
- **CSP and inline scripts**: future UI patches must use module files and event listeners; new `innerHTML` patterns will be blocked by the existing Content Security Policy.

## Numerical accuracy (recent upgrades)

- **Variational flow now uses an exact analytic Jacobian** for the double pendulum (`jacobianDouble`), removing the ~1e-7 error floor that a finite-difference Jacobian imposed on the full Lyapunov spectrum, SALI and FLI. Systems without a closed-form Jacobian fall back to an O(h^2) **central-difference** Jacobian (previously forward-difference). Both are checked against each other in `tests/analytic-derivatives.test.ts`.
- **The canonical implicit-midpoint integrator is now genuinely symplectic**: `hamiltonianGradient` is the exact analytic gradient of H(q,p) rather than a central-difference approximation, so the method integrates the true Hamiltonian vector field.
- **Lyapunov estimates now report uncertainty** (standard error and an approximate 95% CI from the converged per-interval local exponents). The naive SE ignores autocorrelation between renormalization intervals and is a lower bound; a **batched-means / block-bootstrap standard error** (`batchedStandardError`, surfaced as `blockStdError` on the result and CSV export) decorrelates neighbouring intervals for a more honest uncertainty.
- **The analytic Jacobian is validated against an external closed form**, not just another finite difference: linearised about the downward equilibrium, the lower-left block of `jacobianDouble` reproduces the textbook equal-pendulum normal-mode frequencies ПүВІ = (2 Вұ вҲҡ2) g/l to machine precision (`tests/normal-modes.test.ts`).
- **Integrator orders are empirically certified.** `empiricalOrder` (`src/research/convergenceOrder.ts`) measures the convergence order by Richardson self-convergence вҖ” logвӮӮвҖ–y(dt)вҲ’y(dt/2)вҖ–/вҖ–y(dt/2)вҲ’y(dt/4)вҖ– вҖ” confirming eulerвүҲ1, rk2вүҲ2, rk4вүҲ4, gauss2вүҲ4 with no analytic reference (`tests/convergence-order.test.ts`).
- **Structure preservation is empirically certified over long runs.** `energyDriftProfile` (`src/research/structurePreservation.ts`) integrates a conservative system for thousands of periods and classifies the relative-energy drift as _secular_ (a monotone trend, high-RВІ line) or _bounded_ (an oscillation within a band). On the planar double pendulum (~1000 periods) the non-symmetric **rk4 drifts secularly** (RВІ = 0.99999, |drift| growing past 2В·10вҒ»вҒҙ) while the time-symmetric **2-stage Gauss collocation `gauss2` stays bounded** (RВІ вүҲ 0.19, |drift| вүІ 7В·10вҒ»вҒё with no trend) and **implicit-midpoint `hmidpoint`** likewise вҖ” the defining structure-preserving signature, turning "use a symplectic integrator for long conservative runs" into a measured number (`tests/structure-preservation.test.ts`; CLI `npm run research -- drift`). The same `method` selection is exposed on the chain integrators (`SphericalChain({ method: 'gauss2' })`); for the chains in (Оё, Пү) coordinates the bounded-drift mechanism is the method's time-reversibility rather than strict canonical symplecticity (which would need (q, p) coordinates вҖ” the route the canonical implicit-midpoint integrator already takes).
- **The Lyapunov spectrum is checked for Hamiltonian self-consistency.** `analyzeSpectrumConsistency` verifies the conservative-system constraints (ОЈО» вүҲ 0, symplectic pairing О»_i вүҲ вҲ’О»_{nвҲ’1вҲ’i}, two zero exponents) and the verdict is attached to every spectrum result and shown in the Lyapunov tab status line вҖ” an independent gate on the whole tangent-space pipeline rather than a bare number.
- **Lyapunov runs accept a selectable integrator** (defaulting to rk4); a high-order/symplectic reference can be chosen for long conservative runs.
- **Shadowing / reproducibility horizon** (`shadowingHorizon`) quantifies how long a chosen integrator tracks a high-accuracy (gbs, fine-dt) reference before the chaotic divergence exceeds a tolerance вҖ” turning numerical trust into a measured number.

## Chaos diagnostics (research-grade additions)

- **Independent chaos detector: the 0вҖ“1 test** (`zeroOneTest`, GottwaldвҖ“Melbourne). It works on a scalar observable series alone and never touches the Jacobian or a deviation vector, so K вүҲ 1 (chaotic) / K вүҲ 0 (regular) is a genuine cross-validation of the Lyapunov verdict rather than the same tangent-space computation reported twice. Confirmed against periodic/white-noise series and both ends of the double pendulum (`tests/zero-one-test.test.ts`).
- **Covariant Lyapunov vectors (Ginelli algorithm)** (`covariantLyapunovVectors`). These are the true Oseledets stable/unstable directions вҖ” covariant and generally non-orthogonal вҖ” not the orthogonal GramвҖ“Schmidt frame. Correctness is pinned on a linear system whose CLVs are known non-orthogonal eigenvectors (a GramвҖ“Schmidt implementation would fail this), and the QR-diagonal exponents cross-check `lyapunovSpectrum`. A hyperbolicity measure (the minimum angle between expanding and contracting CLVs) is reported (`tests/clv.test.ts`).
- **Fractal exit structure: flip basins, basin entropy, box-counting dimension** (`doublePendulumFlipBasin`, `basinEntropy`, `boxCountingDimension`). The classic double-pendulum flip map is classified, and its boundary is quantified by the Daza et al. basin entropy and a MinkowskiвҖ“Bouligand dimension (вүҲ1.5вҖ“1.6 вҖ” strictly fractal). The entropy/box-counting routines are pure functions of a label grid, so they apply equally to genuine attractor basins (`tests/basin.test.ts`).
- **Recurrence Quantification Analysis (RQA)** (`recurrenceQuantification`, `recurrenceMatrix`; Marwan et al., Phys. Rep. 2007). A second non-variational diagnostic: a bounded observable is time-delay embedded and the recurrence plot is quantified into DET (determinism вҖ” structure vs noise), LAM, Lmax and DIV = 1/Lmax (a finite-size proxy for the largest Lyapunov exponent), ENTR and trapping time. Pinned on a sine (DET вүҲ 1, long diagonals) vs white noise (low DET), and the double-pendulum chaos signature (a regular orbit has far longer diagonals / smaller divergence than a chaotic one) cross-validates the spectrum (`tests/rqa.test.ts`).
- **Finite-Time Lyapunov Exponent (FTLE) fields** (`finiteTimeLyapunov`, `flowMapGradient`, `doublePendulumFtleField`). Пғ_T(xвӮҖ) = (1/T) ln Пғ_max(вҲӮx(T)/вҲӮx(0)) вҖ” the largest singular value of the _finite-time_ flow-map gradient, obtained by propagating the variational state-transition matrix (same analytic Jacobian, no finite-difference floor). Ridges of the field are Lagrangian Coherent Structures, distinct from the Sweep's time-averaged О»вӮҒ. Pinned against the exact `exp(AT)` of linear systems (FTLE and determinant analytically known) and the double-pendulum chaotic-vs-regular separation (`tests/ftle.test.ts`). The single-largest singular value is found by power iteration on MбөҖM.
- **Floquet stability of periodic orbits** (`floquetAnalysis`, `drivenPeriodicOrbit`, `eigenvalues2x2`, `monodromyMatrix`). The monodromy matrix over one period (the same state-transition matrix as the FTLE flow-map gradient) has the Floquet multipliers ПҒ_i as eigenvalues; |ПҒ_i| < 1 вҮ’ a stable orbit. For the driven pendulum a period-1 orbit is found as a fixed point of the stroboscopic map by a 2-D Newton step whose Jacobian is the monodromy itself. Planar (2Г—2) scope, so the eigenvalues are closed-form (no general eigensolver). Pinned on linear oscillators whose monodromy determinant is analytically e^{вҲ«div} вҖ” undamped (unit circle), damped (О |ПҒ| = e^{вҲ’2ОіT}, stable), inverted (a multiplier outside the unit circle, unstable) вҖ” and the driven-pendulum stroboscopic orbit (`tests/floquet.test.ts`). **Now surfaced in the Bifurcation tab** ("Driven Orbit: Floquet & Pitchfork" вҶ’ _Floquet @ A_), branch-traced from A = 0.7 so the multipliers track the symmetric oscillating branch; cross-browser e2e in `e2e/modern-bifurcation-tab.spec.ts`.
- **Numerical continuation + bifurcation detection** (`continueDrivenPeriodicOrbit`, `classifyBifurcation`). Natural-parameter continuation traces the driven-pendulum period-1 orbit across a parameter (warm-starting Newton from the previous orbit) and reports the first stability loss, classified by the Floquet crossing: ПҒ вҶ’ вҲ’1 period-doubling, ПҒ вҶ’ +1 tangent (fold/transcritical/pitchfork вҖ” reported generically), a complex pair through |ПҒ| = 1 NeimarkвҖ“Sacker. Validated on constructed multiplier sets and on the damped driven pendulum, whose symmetric period-1 orbit loses stability via a real multiplier crossing +1 near drive amplitude A вүҲ 1.01 (`tests/continuation.test.ts`). Library API.
- **Pseudo-arclength continuation with fold detection** (`continueArclength`, Keller's method). Natural-parameter continuation stalls at a fold (the branch has a vertical tangent there); pseudo-arclength advances along the branch's arclength, solving the augmented system [G(x,О»)=0; П„В·О”вҲ’ds=0] whose Jacobian (last row П„) stays non-singular through the turning point, so it traces _around_ folds. The Jacobians are finite-differenced, so it is generic over any residual G(x,О») (algebraic, or the stroboscopic fixed-point condition of the driven pendulum). Pinned on branches with closed-form folds: xВІвҲ’О» (one fold at the origin) and xВівҲ’3xвҲ’О» (folds at (1,вҲ’2) and (вҲ’1,2)), staying on the solution curve to ~1e-10 and locating the folds to the step granularity (`tests/arclength.test.ts`). Library API.
- **Every non-variational diagnostic now carries an uncertainty estimate**, extending the Lyapunov block-bootstrap SE to the rest of the toolbox: the 0вҖ“1 test reports a seeded bootstrap SE and percentile 95% CI of the median K (the per-frequency K_c are i.i.d. over random frequencies, so the ordinary bootstrap is the right scheme); RQA reports block-resampled SEs for DET/DIV (contiguous blocks preserve the temporal recurrence structure вҖ” batched-means style вҖ” and cost ~1/k of the O(NВІ) quantification); basin entropy reports the SEM of Sb/Sbb over boxes; and the box-counting dimension reports the regression slope SE, Student-t 95% CI and RВІ of the log-log fit (a low RВІ flags a dimension that should not be trusted). All surfaced in the 0вҖ“1/RQA/Basin tab readouts and status lines (`tests/uncertainty.test.ts`).
- **Literature anchors** (`runLiteratureAnchors`, `npm run validate:literature`, CI job): engine-computed quantities are compared head-to-head against values that exist _outside_ the codebase вҖ” the large-amplitude pendulum period vs 4K(sin(ОёвӮҖ/2))/ПүвӮҖ (complete elliptic integral, agreement < 1e-6), the equal-double-pendulum normal modes (2 вҲ“ вҲҡ2)g/l, the Melnikov A_c by quadrature vs closed form, and the damped driven pendulum period-doubling onset measured from the Floquet multiplier crossing вҲ’1: **A_PD = 1.06637 measured vs 1.0663 published** (Baker & Gollub). Structural checks: A_c < A_PD (the homoclinic tangle precedes the attractor cascade) and the flip-basin boundary dimension is strictly fractal. Report: `reports/literature-anchors.md`; tests: `tests/literature-anchors.test.ts`.
- **Melnikov analysis вҖ” the analytic chaos threshold** (`melnikovCriticalAmplitude`, `melnikovFunction`, `melnikovVerdict`; CLI `npm run research -- melnikov`). For the damped driven pendulum the Melnikov function along the unperturbed separatrix evaluates in closed form, M(П„вӮҖ) = вҲ’8Оҙ + 2ПҖfВ·sech(ПҖО©/2)В·cos(О©П„вӮҖ), giving the critical drive amplitude A_c = (4ОіПүвӮҖ/ПҖ)В·cosh(ПҖПү/(2ПүвӮҖ)) above which a transverse homoclinic intersection (Smale-horseshoe tangle) exists. The closed form is pinned against brute-force Simpson quadrature of the Melnikov integral along the separatrix (вүӨ1e-8), the known value A_c вүҲ 1.0187 for Оі = 0.5, Пү = 2/3, and 0вҖ“1-test physics consistency: regular motion well below A_c, and the literature chaotic preset (A = 1.15) above it (`tests/melnikov.test.ts`). Being first-order perturbation theory, A_c is asymptotically exact only for small damping/forcing; the tangle it predicts yields transient chaos and fractal basin boundaries, with the sustained chaotic attractor appearing somewhat above A_c (the measured period-doubling onset A_PD вүҲ 1.066 indeed sits above A_c вүҲ 1.019 вҖ” the expected ordering).
- **Five of these are first-class Lab tabs**, not just library APIs: the 0вҖ“1 Test (в—җ), Covariant Lyapunov Vectors (вҮҢ), Flip Basins (вқӢ), Recurrence Quantification (в–Ё) and FTLE Field (в©ң) tabs each run their job on the chaos worker (with the same transparent main-thread fallback as the Lyapunov tab) and read the live on-page system controls via `readSystem()`. The worker path is exercised by `tests/chaos-protocol-diagnostics.test.ts` (the same pure `runChaosJob` handler the UI uses) and the tab takeovers by `e2e/modern-chaos-diagnostics.spec.ts`. The rail icons are grouped into Simulate / Analyze / Chaos-diagnostics / Validate clusters (subtle dividers, scroll-safe).

## Spherical chain, symbolic reference, and the mini-paper (v10.30)

- **The spherical double/triple pendulum** (`src/physics/sphericalChain.ts`) is genuine 3D chain dynamics (ball joints, 4/6 DOF), not a camera over planar motion. Its equations are assembled in manipulator form from closed-form Jacobians; validity is pinned by exact reduction to the single spherical pendulum and to the planar chain, by E and L_z conservation in 3D chaos (with 4th-order drift shrinkage under dt-halving вҖ” a wrong RHS would leave a dt-independent floor), and by the independent SymPy symbolic derivation (~1e-14). Known limitation: the (Оё, ПҶ) chart is singular at the poles; it is regularised at |sinОё| = 1e-6, so trajectories that pass _exactly_ through a pole are perturbed at that scale. Damping is per-coordinate rate damping (matching `spherical.ts`), which differs from the planar chain's force-level convention вҖ” planar comparisons are therefore made at Оі = 0. For the **single** spherical pendulum this pole limitation is now removed by an alternative **embedded (Cartesian unit-vector) chart** (`EmbeddedSphericalPendulum`, `src/physics/sphericalEmbedded.ts`): the ambient member of the SВІ atlas, globally regular, with the rod constraint enforced by projection. It agrees with the (Оё, ПҶ) chart to ~2e-10 away from the poles and conserves E and L_z to ~1e-11 through a near-pole passage where the clamped polar chart loses ~30% of the energy (or diverges to NaN) вҖ” `tests/spherical-embedded.test.ts`. **The embedded _chain_ is now implemented too** (`EmbeddedSphericalChain`, `src/physics/sphericalEmbeddedChain.ts`): each link is an independent Cartesian unit vector u_k вҲҲ SВІ with the rod constraints enforced by N Lagrange multipliers (the rod tensions). The decisive structural point is that the kinetic metric in the embedded chart is the _constant_ suffix-mass matrix A_{jk} = l_j l_k S_{max(j,k)}, so there are **no configuration-dependent Coriolis/Christoffel terms** вҖ” all nonlinearity is in the constraint forces, and the per-step multiplier system G О» = c has G = BвҲҳU (Hadamard product of the SPD inverse-metric with the unit-diagonal Gram matrix of the link directions), which is **positive-definite for every configuration by the Schur product theorem**. Unlike the polar chain there is therefore no singular solve and no pole clamp anywhere. Validated by exact reduction to `EmbeddedSphericalPendulum` at N = 1 (RHS component-wise and ~7e-10 over a 3 s trajectory), agreement with the polar `SphericalChain` to ~1.8e-7 away from the poles, E/L_z conservation to ~1e-5 through a near-pole passage where the clamped polar chart drifts > 10Г— more, 4th-order drift shrinkage under dt-halving, and a positive-definite multiplier matrix at a state with a link essentially _at_ the pole (`tests/spherical-embedded-chain.test.ts`). Library API; the natural future step is to surface it in the 3D lab.
- **The SymPy reference** (`npm run validate:sympy`) checks the _derivation_ (RHS component-wise at random states), deliberately not trajectories вҖ” the SciPy DOP853 comparison already covers integration. It validates conservative dynamics only (Оі = 0), since the damping conventions are engine choices that a Lagrangian derivation does not fix.
- **The mini-paper** (`paper/`) measures A_PD(Оі) for the _primary oscillating attractor branch_, warm-started in A. At the lowest dampings the phase space is multistable and basin-capture transitions can occur below the verified doubling (observed at Оі = 0.15); the quoted onsets are specifically Floquet-verified ПҒ = вҲ’1 crossings. The Melnikov comparison uses the first-order closed form; its breakdown at strong damping is part of the paper's result, not an engine defect.

## Remaining items

- Direct `file://` use is supported: download the versioned `pendulum-lab-standalone-v*.html` GitHub Release asset (produced by `npm run build:standalone`) and double-click it to run the full Lab. The generated file is intentionally excluded from Git history; `standalone-manifest.json` pins its release hash. The dev/HMR shell lives in `app.html`; serving via Vite remains the path for development. Over `file://`, browsers may block module workers; physics, chaos, and Expansion jobs emit `pendulum-lab:worker-fallback` and a toast before using the main thread.
- Features intentionally **dropped** in the modern rewrite (vs. the old legacy lab): interpolated render and trail color-mode selection (cosmetic), the NaN-recovery overlay (the modern integrators are stable; a fault simply pauses), and some developer-hub-only actions. The submission-manifest export now captures a **live snapshot** (UI controls + running sim state + diagnostics) via `currentSnapshot()`, including the `index.exportSubmissionManifest` command.
- The `runtime/LegacyBridge` and `runtime/IndexPhysicsBridge` shims have been **deleted** (archived). Removing the bridge also fixed a real bug: it registered a second Space/R key handler alongside the modern shell's, so pressing Space toggled pause _twice_ (a no-op). The public scripting surface (`window.PendulumLabIndex`) and the default command set now install directly from `main.ts`.
- The benchmark script can launch the local Vite server automatically, but true original-vs-candidate comparisons still require distinct `ORIGINAL_URL` and `CANDIDATE_URL` inputs.
- The triple pendulum is now cross-validated against the independent SciPy reference (see the external cross-validation bullet below): a regular small-angle orbit agrees to ~6e-14 over 20 s and a strongly chaotic orbit to ~4e-8 at T = 8 s (the tolerance floor amplified by e^{О»вӮҒt}), with deliberately asymmetric masses/lengths so index-transposition bugs cannot cancel.
- Covariant Lyapunov vectors (Ginelli) and finite-time Lyapunov (FTLE) fields are now implemented and surfaced as Lab tabs; the full spectrum, CLVs and variational FTLE are CPU-side publication oracles. GPU acceleration has an executable promotion contract (`compareLyapunovSpectrumAcceleration`, `compareClvAcceleration`, `compareFtleFieldAcceleration`) and must pass that contract before replacing a CPU result. The 0вҖ“1 test, CLV, basin-entropy, RQA and FTLE diagnostics are dedicated Lab UI tabs (in addition to their library/test APIs); Floquet stability and the symmetry-breaking pitchfork are now surfaced in the **Bifurcation tab** (in addition to the library and CLI).
- A **double-double (~31-digit) extended-precision reference** is now available (`src/validation/doubleDouble.ts`): error-free-transform arithmetic (two-sum / two-product, DekkerвҖ“Knuth), double-double division, and double-double sin/cos (argument-reduced Taylor series, accurate to dd precision вҖ” sinВІ+cosВІ = 1 holds to ~1e-29 where float64 manages only ~1e-16), plus double-double RK4 and a time-symmetric StГ¶rmerвҖ“Verlet integrator. A symmetric forward/backward round trip shows ~17 orders of magnitude less round-off than float64, and the double-double RK4 acts as an exact-arithmetic reference that measures the float64 integrator's accumulated round-off (`tests/double-double.test.ts`).
- The double-double arithmetic is now applied to the **double pendulum itself** (`src/validation/doubleDoublePendulum.ts`): `ddDoublePendulumRhs` re-expresses the mass-matrix solve of `rhsDouble` in double-double and is validated component-wise against the trusted float64 `rhsDouble` to ~1e-15. Using it as the ground truth, the float64 RK4 orbit is shown to track the true trajectory only briefly (~1e-14 at T = 2 s) before the chaotic dynamics amplify its round-off by orders of magnitude (~1e-7 at T = 10 s) вҖ” a predictability limit an _in-precision_ reference cannot expose about itself (`tests/double-double-pendulum.test.ts`). This closes the earlier gap where the pendulum's only reference was a high-accuracy in-precision orbit. (The reference is conservative/undamped.)
- **External cross-validation against SciPy and Julia is automated**: SciPy DOP853 independently covers double/triple systems, while pinned Julia 1.10.11 + OrdinaryDiffEq Vern9 is a required reference gate. MATLAB remains optional.
- A **Wada-property candidate test** (`wadaCandidate`, grid method of DazaвҖ“WagemakersвҖ“SanjuГЎn) classifies what fraction of basin-boundary cells touch вүҘ 3 basins; it is reported in the Basin tab status line and refinable from the CLI (`npm run research -- wada --n 150`). At a single resolution it is _evidence_, not proof вҖ” the verdict field is deliberately named `wadaCandidate`.
- A **headless research CLI** (`npm run research -- <command>`) runs every chaos job (Lyapunov, spectrum, 0вҖ“1, RQA, FTLE, basin, study point) through the very same `runChaosJob` handler the app's worker uses, plus the periodic-orbit finder and continuation вҖ” so the browser UI, the worker, the unit tests and the CLI all exercise one code path.
